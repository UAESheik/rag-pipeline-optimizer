# RAG 优化器最终报告（本轮运行结果）

## 1. 本次运行结论
本次实验已经完整跑通 Case1（有监督）和 Case2（弱监督）两条优化链路，并输出了最优配置、逐 query 诊断、指标相关性检查以及 Pareto 前沿文件。整体上，系统已具备可展示、可解释、可回溯的交付形态。

## 2. Case1 结果摘要
- 最优配置：`cfg_00003`
- 检索器：`dense`
- embedding：`e5-base`
- 分块：`sentence / size=512`
- 重排：`True`
- 查询改写：`True`
- 训练分数：`0.6276`
- 保留集分数：`0.6178`
- 过拟合差值：`0.0098`

### 2.1 结果解读
Case1 的训练集与保留集分数接近，说明配置较稳定，没有明显过拟合。该场景下，答案质量与证据支撑的综合表现最好，核心提升来自于 `dense + rewrite + rerank` 的组合。

## 3. Case2 结果摘要
- 最优配置：`cfg_00019`
- 检索器：`hybrid`
- embedding：`e5-base`
- 分块：`token / size=256`
- 重排：`False`
- 查询改写：`False`
- metadata 过滤：`True`
- 生成模式：`citation_first`
- 训练分数：`0.4928`
- 保留集分数：`0.6096`
- 过拟合差值：`-0.1168`

### 3.1 结果解读
Case2 的保留集分数明显高于训练集，说明弱监督场景下样本波动较大，但系统仍然能找到一组偏向证据覆盖和引用一致性的稳健配置。该结果也说明：弱监督优化更依赖检索证据质量和生成约束，而不是单纯追求答案表面相似度。

## 4. 关键对比
| 对比维度 | Case1（有监督） | Case2（弱监督） |
|---|---|---|
| 最优配置 | `cfg_00003` | `cfg_00019` |
| 检索器 | dense | hybrid |
| 分块策略 | sentence / 512 | token / 256 |

| 生成策略 | concise | citation_first |

## 5. 这次运行说明了什么
- **Case1 更适合“答案一致性 + 证据支撑”**，rewrite 和 rerank 的收益更明显。
- **Case2 更适合“证据覆盖 + 引用绑定”**，citation-first 和更保守的检索策略更稳定。
- `run_summary.csv` 与 `per_query_diagnostics.csv` 已经能展示每次 trial 的技术效果，而不只是最终分数。
- `metric_sanity_case2.csv` 说明弱监督指标已经能反映一部分优化方向，但 groundedness 与上下文相关性仍有改进空间。
